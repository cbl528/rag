RAG 智能体 — 前端 UI 风格设计说明书

一、设计哲学

▎ "少即是多，让内容呼吸"

以 Apple 的 Human Interface Guidelines 为底层逻辑，承载 ChatGPT/Gemini
级别的对话体验。不做炫技式的视觉轰炸，而是通过克制的色彩、精确的间距、自然的动效，让用户感觉在使用一件"数字工艺品"。

三、字体排印（Typography）

3.1 字体家族

font-family: -apple-system, BlinkMacSystemFont, "SF Pro Display",
"PingFang SC", "Hiragino Sans GB", "Microsoft YaHei",
sans-serif;

- 西文优先使用 SF Pro（Apple 系统字体栈）
- 中文使用 苹方 降级至 微软雅黑

四、空间与布局（Spacing）

4.1 核心网格

基准单位 8px，所有间距为 8 的倍数：

- 4px — 紧凑元素间距（图标与文字）
- 8px — 基础内边距
- 16px — 消息气泡内边距、卡片 padding
- 24px — 区域间距、对话列表 item 间距
- 32px — 大区块间距
- 48px — 页头 / 页尾留白

4.2 布局结构

│          │  │      水平居中                  │
│          │  │                               │

- 消息气泡最大宽度：90% of 对话区宽度（约 690px）

5.5 Markdown 内容渲染

- 标题：使用系统字重，h1 → 22px/700, h2 → 18px/600
- 代码块：深色背景 #1C1C1E，圆角 12px，带语言标签和复制按钮
- 引用：左侧 3px Accent Blue 竖线，文字次要色
- 列表：圆点使用 Accent 色
- 链接：Accent Blue，hover 时加下划线
- 表格：极简线条，斑马纹行背景

六、动效系统（Animation）

6.1 核心原则

▎ 动效是信息的传递，不是装饰。

所有动画使用 Apple 风格缓动曲线：

/* 标准缓出 — 出现、展开 */
--ease-out: cubic-bezier(0.25, 0.1, 0.25, 1.0);

/* 弹性缓出 — 重要元素入场 */
--spring: cubic-bezier(0.34, 1.56, 0.64, 1);

/* 平滑 — 颜色、背景过渡 */
--ease-smooth: cubic-bezier(0.4, 0, 0.2, 1);

6.2 具体动画

┌─────────────────┬────────────────────────────┬────────────────────┐
│      场景       │            动画            │        时长        │
├─────────────────┼────────────────────────────┼────────────────────┤
│ 消息进入        │ 从下向上滑入 + opacity 0→1 │ 300ms, spring      │
├─────────────────┼────────────────────────────┼────────────────────┤
│ AI 思考中       │ 三个原点依次跳动（弹跳点） │ 600ms 循环         │
├─────────────────┼────────────────────────────┼────────────────────┤
│ 侧边栏展开/收起 │ 宽度变化 + 内容淡入        │ 250ms, ease-out    │
├─────────────────┼────────────────────────────┼────────────────────┤
│ 按钮 hover      │ scale 1.05 + 微阴影增强    │ 150ms, ease-smooth │
├─────────────────┼────────────────────────────┼────────────────────┤
│ 输入框 focus    │ 边框变色 + 外发光          │ 200ms, ease-smooth │
├─────────────────┼────────────────────────────┼────────────────────┤
│ 页面切换        │ 交叉淡入淡出               │ 200ms              │
├─────────────────┼────────────────────────────┼────────────────────┤
│ 滚动            │ smooth scroll behavior     │ —                  │
└─────────────────┴────────────────────────────┴────────────────────┘

七、图标系统

- 使用 SF Symbols 风格的线性图标（推荐 Lucide Icons (https://lucide.dev)）
- 统一 1.5px 描边宽度
- 大小：侧边栏 20px，功能按钮 18px，发送按钮 22px
- 色彩跟随文字色，hover 时变为 Accent Blue

九、技术落地建议

结合你已有的 Vue 3 + Vite 技术栈，建议：

┌──────────┬─────────────────────────────────────────────────────┐
│   层面   │                      推荐方案                       │
├──────────┼─────────────────────────────────────────────────────┤
│ CSS 框架 │ Tailwind CSS（配合自定义 Apple 风格 design tokens） │
├──────────┼─────────────────────────────────────────────────────┤
│ 组件库   │ 不引入重型 UI 库（保持原生感），按需自建组件        │
├──────────┼─────────────────────────────────────────────────────┤
│ Markdown │ marked + highlight.js（代码高亮）                   │
├──────────┼─────────────────────────────────────────────────────┤
│ 图标     │ lucide-vue-next（SF Symbols 风格）                  │
├──────────┼─────────────────────────────────────────────────────┤
│ 动效     │ CSS transition/animation + Vue <Transition>         │
├──────────┼─────────────────────────────────────────────────────┤
│ 深色模式 │ CSS 变量 + prefers-color-scheme + 手动 toggle       │
└──────────┴─────────────────────────────────────────────────────┘

  ---
这就是完整的风格设计说明书。你可以从以下几个角度来审视是否合适：

1. 色彩是否够"Apple"？ — 大面积灰白底 + 蓝色点缀，克制不跳跃
2. 间距是否有呼吸感？ — 8px 基准网格，内容不拥挤
3. 动画是否自然？ — 用 spring 曲线而非 linear
4. 深色模式是否"真黑"？ — OLED 纯黑底，不是深灰
